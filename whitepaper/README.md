## Contribution Guidelines

To contribute to the whitepaper:

1. Fork the repository
2. Create a branch: `whitepaper-improvement-<description>`
3. Make changes with clear rationale
4. Submit PR with:
   - Summary of changes
   - Motivation
   - Impact assessment
5. Await Elder review + community feedback

All contributions must:
- Maintain integrity-first principles
- Improve clarity or accuracy
- Pass GI-Sim evaluation (conceptually)
- Preserve constitutional invariants
